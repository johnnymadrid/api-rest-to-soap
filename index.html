API REST To SOAP

Although both technologies are related to web services. Here you will find an implementation to accessing SOAP Services using an endpoint as a REST service. The main idea to use this kind of implementation is to maintain easily the accessibility SOAP services, which developers are commonly struggling to understand large numbers of XML tags, a structure of a tree with namespaces privately. In other words, this implementation of the REST service makes an understandable start of a point to use another service, which has been implemented in other technology, using JSON messages.

What is JSON?

Although it is not the goal to explain extensibly JSON and benefits about this language, below are the short definition to start;

  • JSON means JavaScript Object Notation,
  • It is open standard format and "human readable text to transmit data composed of attribute value-key pairs,
  • Related to the official Internet media type application/json,
  • It is an independent language using Javascript format but is text,
  • Extension file is .json

SOAP Service concerning

The chosen example for this API implemented is the SOAP Service - TShirt Service published in the following URL http://tshirt-service.cloudhub.io/. Its WSDL definition is here.

The SOAP Service is part of the extensive services published in the integration platform service in Mulesoft (CloudHub).
